# STAT 3302 PROJECT - Bike Sharing Group A
Statistical Analysis on Bike Sharing/Rental data for STAT 3302 at the Ohio State University. Reports explore the data using a Poisson Regression model to model the rate of Bike Rentals per hour based on a certain set of predictors (weather conditions + nature of calender days + user resgistration to bike sharing service + seasons)


## Languages/Tools Used
- **Language:** R (via Quarto on RStudio IDE)
- **Libraries:** tidyverse, ggthemes
- **Data:** (http://capitalbikeshare.com/system-data)

## Instructions on contributions with Git
### Prerequisites/Getting Started
1. Install the latest version of git on your local system + make a GitHub account
2. Clone the repo: `git clone {url_from_the_github_repo}`

### Pulling changes down (Fetching)
1. Open the directory your project is in RStudio via "Session" -> "Set Working Directory"
2. Open up "Terminal" in Rstudio (should be next to the console). Type `git branch` and check if you are on `main`
3. Pull changes command in the terminal: `pull origin main`

## Push changes up to the repository (Committing)
1. Open the directory your project is in RStudio via "Session" -> "Set Working Directory"
2. Open up "Terminal" in Rstudio (should be next to the console). Type `git branch` and check if you are on `main`
3. Make your changes (work in bursts of short changes and commiting) and once your ready, enter `git add .` in your terminal
4. Commit changes locally by `git commit -m "{your_commit_message}" `
5. Push changes to the actual repository in GitHub by `git push origin main`
